# 프로젝트 개발 순서

1. Issue 생성
2. Branch 생성
    - Branch명 : (#이슈번호)-(feature/fix/...)/(설명)
3. 기능 개발
4. Push
    - Commit명 : (FEAT/FIX/REFACTOR/...): (#이슈번호) (설명)
5. Merge
   - Merge명 : (FEAT/FIX/REFACTOR/...): (#이슈번호) (설명)
6. Issue 닫기

- FEAT : 기능 추가
- FIX : 버그 수정
- REFACTOR : 코드 리팩토링
- TEST : 테스트 코드 추가
- CHORE : 빌드 업무 수정, 패키지 매니저 수정
- DOCS : 문서 수정
